## 核心目标

1. ✅ 每次执行生成独立日志文件（与截图同目录）
2. ✅ 按 logger 等级梳理全部代码，补全日志
3. ✅ CLI 模式日志与文件日志保持同等级别
4. ✅ 日志分级：INFO（日常）、DEBUG（调试）

## 关键改进

### 结果目录结构

**改进前**：
```
result/
└── result_20260321_100000.xml
└── screenshots/
```

**改进后**：
```
result/
└── run_20260403_153045/
    ├── result.xml
    ├── execution.log          ← 新增
    └── screenshots/
```

### 日志输出

**改进前**：
- 大量 `print()` 直接输出
- 日志与结果分离
- 无法追溯历史执行

**改进后**：
- 统一使用 logger
- 日志与结果在同一目录
- 双通道输出（终端 + 文件）
- 支持等级控制（--log-level）
