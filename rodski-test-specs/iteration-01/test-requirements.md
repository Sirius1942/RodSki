# 迭代 01 - 视觉定位功能测试需求

**版本**: v1.0
**日期**: 2026-03-27
**对应开发迭代**: iteration-01

## 测试目标

验证基于 OmniParser+LLM 的视觉定位功能在 Web 和 Desktop 平台的正确性、稳定性和性能。

## 测试范围

### 1. Web 平台视觉定位

#### 1.1 vision 定位器（语义描述）
- 通过文字描述定位按钮
- 通过文字描述定位输入框
- 通过文字描述定位图标

#### 1.2 ocr 定位器（文字识别）
- 定位中文文字
- 定位英文文字
- 定位混合文字

#### 1.3 vision_bbox 定位器（坐标）
- 直接坐标点击
- 直接坐标输入

#### 1.4 混合定位器
- 传统定位器 + 视觉定位器降级
- 多定位器优先级切换

### 2. Desktop 平台视觉定位

#### 2.1 应用启动
- macOS 应用启动（launch 关键字）
- 窗口切换

#### 2.2 桌面操作
- 文本编辑器输入
- 按钮点击
- 菜单操作

### 3. 关键字统一性

验证 type/verify/launch 关键字在不同平台的一致性。

## 测试用例设计

### TC01: Web 视觉定位 - 登录场景

**前置条件**: 浏览器可访问测试页面

**测试步骤**:
1. 打开登录页面
2. 使用 ocr 定位用户名输入框，输入用户名
3. 使用 ocr 定位密码输入框，输入密码
4. 使用 vision 定位登录按钮，点击
5. 验证登录成功

**预期结果**: 所有步骤成功执行

### TC02: Web 混合定位器降级

**前置条件**: 浏览器可访问测试页面

**测试步骤**:
1. 定义元素包含 id 和 ocr 两个定位器
2. 执行点击操作
3. 验证优先使用 id 定位器

**预期结果**: 按优先级使用定位器

### TC03: Desktop 应用操作 - 文本编辑

**前置条件**: macOS 系统

**测试步骤**:
1. 使用 launch 启动 TextEdit
2. 使用 ocr 定位编辑区域
3. 输入文本
4. 验证文本内容

**预期结果**: 文本正确输入

### TC04: vision_bbox 坐标定位

**前置条件**: 已知元素坐标

**测试步骤**:
1. 使用 vision_bbox 定位元素
2. 执行点击操作

**预期结果**: 点击成功

## 测试数据

- 测试页面: 本地 demo 页面（包含登录表单）
- 测试应用: TextEdit (macOS)
- 测试文本: 中英文混合

## 测试环境

- Python 3.9+
- macOS 11+
- OmniParser 服务运行中
- LLM API 可用

## 成功标准

- 所有测试用例通过率 100%
- 视觉定位响应时间 < 3s
- 无严重异常或崩溃
