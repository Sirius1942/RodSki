# 版本号命名规范

**创建日期**: 2026-04-13

---

## 格式

```
大版本.功能版本.Bug修复
  X   .   Y   .  Z
```

## 规则

| 位 | 含义 | 何时递增 | 示例 |
|----|------|---------|------|
| **X（大版本）** | 架构级变更、不兼容的 Breaking Change | 核心架构重写、数据格式不兼容、API 全面变化 | v4 → v5: Excel → XML 迁移 |
| **Y（功能版本）** | 新功能、功能增强、重构、文档大改 | 每个迭代交付一个功能版本 | v5.3 → v5.4: 契约统一 |
| **Z（Bug 修复）** | 缺陷修复、小补丁、文档小修 | 不改功能，只修问题 | v5.3.1 → v5.3.2: 配置清理 |

## 递增原则

1. **Z 递增**：Bug 修复、热修补、文档纠错 — 不改功能行为
2. **Y 递增**：新功能、功能增强、重构、契约变更 — Z 归零
3. **X 递增**：架构级 Breaking Change、核心不兼容变更 — Y 和 Z 归零

## 当前版本

- 最新发布：**v5.3.2**
- 下一个功能迭代：**v5.4.0**

## 分支与版本对应

| 迭代类型 | 分支命名 | 版本示例 |
|---------|---------|---------|
| 功能迭代 | `release/v5.4.0` | v5.4.0 |
| Bug 修复 | `fix/v5.4.1-描述` | v5.4.1 |
| 大版本 | `release/v6.0.0` | v6.0.0 |

## 判断标准

- "这次改了功能吗？" → 是 → Y+1
- "这次只修了 Bug？" → 是 → Z+1
- "这次改了核心架构，旧用法不兼容了？" → 是 → X+1
