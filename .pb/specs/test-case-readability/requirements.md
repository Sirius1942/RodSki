# RodSki 测试用例可读性提升需求文档

**版本**: v1.0
**日期**: 2026-03-31
**状态**: 待开发

---

## 1. 背景与目标

### 1.1 问题描述

当前 RodSki 测试用例以 XML 格式编写，虽然结构化程度高，但对于非技术人员或业务方来说，**直接阅读 XML 用例的可读性较差**：

- XML 标签和属性对业务人员不直观
- 难以快速理解用例的业务意图
- 测试步骤缺乏自然语言描述
- 用例执行结果报告不够友好

### 1.2 目标

提升 RodSki 测试用例的可读性，使得：
1. **技术人员**：更快速理解用例结构和执行逻辑
2. **业务人员**：能看懂用例的业务意图和测试步骤
3. **管理者**：能通过友好的报告了解测试覆盖情况

### 2.2 业务标签系统

支持在用例中添加业务元数据：

```xml
<case id="cassmall_inquiry_publish" 
      title="小李发布询价"
      purpose="验证维修厂用户能够成功发布询价单"
      priority="P0"
      tags="询价,核心流程,冒烟测试"
      component_type="界面">
```

**标签说明**：
| 属性 | 说明 | 示例 |
|------|------|------|
| `purpose` | 用例目的描述 | 验证用户能成功登录 |
| `priority` | 优先级 | P0/P1/P2/P3 |
| `tags` | 业务标签 | 询价,核心流程 |
| `component_type` | 组件类型 | 界面/接口/性能 |

### 2.4 自然语言步骤描述

支持在 model.xml 中为元素添加描述：

```xml
<model name="InquiryPage" type="web">
    <element name="vin">
        <type>input</type>
        <location type="id">vin</location>
        <description>车辆识别代号输入框</description>
    </element>
    <element name="submitBtn">
        <type>button</type>
        <location type="text">发布询价</location>
        <description>点击后提交询价单到系统</description>
    </element>
</model>
```

## 3. 验收标准

| 功能 | 验收条件 |
|------|---------|
| explain 命令 | 输出格式友好，包含所有业务元数据 |
| 业务标签解析 | XML 标签能被正确解析和展示 |
| HTML 报告 | 能生成带样式的 HTML 报告 |
| 元素描述 | model.xml 中的 description 能被显示 |

## 4. 技术实现路径

### 4.1 CLI 增强
- 扩展 `cli_main.py explain` 命令
- 支持 `--format` 参数（text/markdown/html）

### 4.2 报告生成
- 新增 `rodski report` 命令
- 生成格式化 HTML 报告
- 支持截图嵌入

### 4.3 Model 扩展
- 在 schema 中支持 `description` 字段
- 解析器支持新字段读取
